我有一个邪恶的想法，实在挥之不去，没办法就开个项目吧

计划将面向对象思想添加到C#Light，让C#Light变得更强大，能做更多事。
让C#Light可以写自己的脚本类型，脚本函数
完全不拒绝任何特性，纯为了邪恶而存在，只为了让脚本更接近C#本尊。
目前计划添加的特性有

1.脚本定义类型
2.支持using
3.完全脚本驱动的代码框架
4.char 类型 byte类型
5.array初始化
6.get set 语法定义
7.委托的支持
8.匿名函数的支持
9.编译时自动添加类型注册(AOT会死的很难看,所以也可能取消，兼容AOT是为了保障IOS这样的强制AOT平台)
10.模板类型支持(AOT会死的很难看，也可能取消)

由于这个想法完全违反了创立C#Light项目时的初衷，故使用完全相反的名称

C#Evil
